## Про проект

Вихідні дані — три Excel-файли різних форматів із продажами за місяць по 13 магазинах. Загалом близько 8 500 рядків з транзакціями. Дані прийшли у сирому вигляді: без категорій, без брендів, без смаків — тільки назва товару, кількість і сума.

Завдання: привести дані до ладу, побудувати довідник товарів і зробити аналітику в Power BI.

**Що зроблено:**
- написано 4 Python-скрипти для очистки, категоризації та валідації даних
- категоризовано 822 унікальні SKU по 6 категоріях і 22 підкатегоріях
- з назв товарів regex-ом витягнуто вагу, смак і бренд
- довідник перевірено через Claude API батчами по 25 товарів
- частина полів доопрацьована вручну де автоматика не справилась
- в Power BI побудовано модель і три інтерактивні дашборди

## Інсайти

**Магазин 13 — лідер з відривом.** Виручка 1,61М, зелений по більшості категорій на тепловій карті, єдиний впевнено в зоні Зірок на scatter plot. Варто розібратись що саме там працює і перенести досвід на інші магазини.

**Магазин 5 — аномалія.** По тепловій карті покриття слабке, переважно середні і мінімальні оцінки по категоріях. При цьому виручка 1,07М і магазин потрапляє в зону Зірок. Схоже результат тягне вузька кількість SKU, а решта асортименту не працює. Варто перевірити чи малий асортимент в магазині або просто товар не продається.

**М'ясні снеки — прихований потенціал.** Топ-1 SKU по виручці за місяць дає 203К грн. При цьому вся категорія М'ясних снеків займає лише 10,8% від загальної виручки. Якщо один товар тягне таку суму, категорія має потенціал значно більший за поточний. Варто розширити асортимент саме тут.

**Група C, 339 SKU.** 339 позицій генерують разом 612К грн — менше 5% від загальної виручки. Вони займають місце в замовленнях і на полиці. Не обов'язково прибирати всі — але потрібне детальне опрацювання: які товари виправдані суміжним попитом, які просто висять без руху.

**Товари без бренду в топі.** Дві позиції горіхів без вказаного виробника — обидві в топ-10 SKU за виручкою. Або власна торгова марка, або постачальник просто не заведений в систему. Варто уточнити.

**Концентрація на одній категорії.** Снеки дають 55,4% виручки. Для зменшення залежності варто поступово нарощувати частку М'ясних снеків і Горіхів.

## Пайплайн

```
3 Excel-файли різних форматів (~8500 рядків)
        |
        v
01_categorize_products.py
    - об'єднання таблиць
    - категоризація: 6 категорій, 22 підкатегорії
    - regex: вага і смак з назви товару
    - бібліотека брендів: визначення ТМ
        |
        v
02_update_master.py
    - створення довідника якщо не існує
    - заповнює тільки порожні клітинки
    - ручні правки зберігаються при повторному запуску
        |
        v
03_extract_brands.py
    - витягує кандидатів на бренди з кінця назви товару
    - виводить топ-100 з частотою для ручного перегляду
        |
        v
04_validate_with_api.py
    - 822 товари батчами по 25 через Claude API
    - перевіряє категорію, підкатегорію, ТМ, смак
    - результат: звіт з колонками OK / Помилка + пояснення
        |
        v
Ручне доопрацювання directory_master_final.xlsx
        |
        v
Power BI: імпорт, зв'язок many-to-one по колонці "товар", дашборди
```
